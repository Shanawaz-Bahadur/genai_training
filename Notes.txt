==> Video Seven - Structured output

LLM returning the output in the well-defined format(e.g, JSON format), rather than free format text

Q) Why it is important to store the LLM output in structured format?
Ans) 1. Data Extraction 
     2. API Building (getting response in JSON format makes it easy to integrate it anywhere)
     3. Agents (Agent -> prompts -> (It inputs the Natural language text)LLM(It responds with the structured output) -> Tools (It expects the structure input))

LLM can generate 2 types of output - 1. structured output(Func - with_structured_output()) and 2. Unstructured output(Func - Output parser[To make it structured])

    i. with_structured_output() (before using this we need to define the data_format of the output)
        a. TypedDict (It defined which keys will be used with there data types with other attributes such as Annotated, Optional, Literal, but the drawback of typedDict is it doesn't give any error if the data types is not followed, so to bypass this drawback we use Pydantic)
        (When to use: Only need type hint, Don't need Validation)

        b. Pydantic (It is a Data Validation and Data parsing library for Python, it ensure that the data you work with is correct, structured and type-safe)
        (When to use: Data Validation, Default Values, Automatic type conversion)

        c. json_schema (As JSON is an universal Data format, if we are using JS in frontend and Python in BE then we can make use of json_schema to make it compatible in both cases)
        (When to use: A Standard JSON format, Don't want to import Python libraries(Pydantic))

    ii. Output Parser - In Langchain it helps convert raw LLM response into structured formats like JSON, CSV, Pydantic models and more. They ensure consistency, Validation, and ease of use in application

        a.StrOutputParser() - It is used to parse the output of a LLM and return it as a plain string

            Response from LLM -> StrOutputParser -> Return only the String(to avoid making use of result.content and it is used to avoid metadata)

            Q) If we have result.content available then why we need to use StrOutputParser()?
            Ans) It is mainly used to make use of chains, as Without it we need to 

        b. JSONOutputParser() - It is used to parse the output of a LLM and return it as a JSON, but without any structured output schema, thats why we move on to the next parser - StructuredOutputParser()

        c. StructuredOutputParser() - It is an output parser in Langchain that helps extract structured JSON data from LLM responses based on predefined field schema, but in this we cannot apply the validation thats why we move to the next parser - PydanticOutputParser()

        d. PydanticOutputParser() -  It is a structured output parser in Langchain that used Pydantic models to enforce schema validation when processing LLM response
 
Video Eight ==> Chains(Using Pipe operator("|"))

As without chains before we need to invoke the prompts and model separately to get the response, but after chains we can directly provide it in the single line (e.g prompt | mmodel |parser)

    1. Sequential Chains (Prompt -> LLM -> parser(Optional))
    2. parellel Chains (Can be designed with the help of RunnableParallel, it is used when we want to have the response from 2 different models parallely (e.g from any study Document I want to generate the notes from one LLM and Quiz from other LLM))
    3. Conditional Chains (Can be designed with the help of Runnablebranch, it is used when we want to some chain if some condition is satisfied or not, (e.g from the feedback if the feedback is Positive then send the thanking response and if the feedback is Negative then send the email to the user))
